Overview

Scale is the abstract projection of your domain specific data value boundaries to screen pixel range. Generally speaking, it lets you to convert any data value into pixel based coordinates on the screen and vice versa. Because of that scales are extremely handy when you build your own Renderer. Nova scale is an extended version of D3 Scale.

And, they are also used for tick generation and tick label formatting. But, in addition to formatting for tick labels, a scale can have the whole set of Formatters to be used for various purposes.

Principles

Every scale is defined by its domain and range. The range will be set by the chart itself and will represent the amount of screen space that is available for rendering. Meanwhile, the domain is controlled by a data provider (it's you) and allows for plenty of possibilities. Some of those possiblities are described on the dedicated Domains page.

Usage

As a user of the chart framework you will usually need to provide a couple of scales with your IChartSeries. Don't know how to do it? You may find it helpful to start with the Chart Basics page first.

But, if you're ready to create a scale, just instantiate any class that implements the IScale interface. Let's use LinearScale as an example: 

{{ scaleDefinition }}

Scale ID

Scales are reusable and can be shared between multiple charts. Scales are identified by the value of their id field. It can be passed as a parameter on scale instantiation, otherwise it will be autogenerated. 

{{ scaleWithId }}
