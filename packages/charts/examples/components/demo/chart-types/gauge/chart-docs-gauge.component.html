Gauges

Gauges can be an effective way to display a single metric value in visual form. And, Nova Charts provides the ability to create two types of gauges (donut and linear) with the assistance of a utility class called GaugeUtil. Three key methods provided by the GaugeUtil will get you most of the way towards a functioning gauge:

  • createChartAssist: can be used to generate a ChartAssist that's pre-configured based on the specified IGaugeConfig and GaugeMode arguments. The ChartAssist handles any necessary pre-processing of the data series to a format that can be used for displaying each of the supported gauge modes.
  • assembleSeriesSet: assembles a gauge series set with all of the scales, renderers, accessors, etc. needed for creating a gauge visualization based on the specified IGaugeConfig and GaugeMode arguments.
  • update: updates a gauge series set using the specified IChartAssistSeries array and IGaugeConfig arguments.

Basic Steps

  1. Create a ChartAssist using GaugeUtil's createChartAssist function.
  2. Update the ChartAssist's associated chart instance with a series set generated by the GaugeUtil's assembleSeriesSet function.
  3. Optionally, at any time after gauge instantiation, update the gauge with a new value, max value, or other configuration changes using the GaugeUtil's update function.

Donut Gauge

A donut gauge is a specialized donut chart with data formatted in a way that results in a gauge-like visualization. It's well-suited for use cases in which there's ample room to display a relatively large visualization. For use cases with a more compact display area, consider the linear gauge whose documentation can be found further down on this page.

Thresholds

To add thresholds to a gauge, specify a value of type IGaugeThresholdsConfig on the thresholds property of the IGaugeConfig. For an easy way to assemble an IGaugeThresholdsConfig object, GaugeUtil provides a createStandardThresholdsConfig function. The definitions property of the function's output includes standard threshold definitions for "warning" and "critical" levels. You can either use the createStandardThresholdsConfig convenience function or construct your own threshold configuration with as many or few threshold definitions as you need.

Definitions

The definitions property of IGaugeThresholdsConfig is organized with an id-to-threshold mapping for easy access. Each threshold definition is of type IGaugeThresholdDef and has an id property with the same value used in the mapping. The id's for common thresholds are standardized in the StandardGaugeThresholdId enum which has Warning and Critical members. Additional properties of the threshold definition include value, color, and enabled.

Colors

As mentioned above, the IGaugeThresholdsConfig return value of the createStandardThresholdsConfig function has a definitions property with a standard set of threshold definitions. Each of these definitions has a color property that's used as the threshold's display color when it's triggered. By default, the function sets the warning threshold to yellow and the critical to red. You can modify the display colors for these and any custom thresholds using this property.

Labels

When threshold definitions are provided on the thresholds property of the IGaugeConfig, the GaugeUtil's createChartAssist function automatically adds a DonutGaugeLabelsPlugin instance to the chart which displays labels for the threshold markers on mouse hover. If you'd like to provide a custom formatter for the labels, take a look at the label customization example on this page for details on how to do that.

Note: If the display strings for your labels are too long to fit within the default grid margins, you can specify a custom clearance value in pixels on the clearance property of the IGaugeConfig's labels property. This will ensure that the grid margins are at minimum the clearance value specified. For the donut gauge, the clearance is applied to all margins; for the linear gauge, the clearance is applied only to the side on which the labels appear.

Reversed Thresholds

The default behavior of a Nova gauge is to treat quantity values that are greater than or equal to the threshold values as being triggered. If you want to reverse that and have the gauge treat quantity values that are less than or equal to the threshold values as triggered, set the reversed property on the IGaugeThresholdsConfig object to true.

Enabling/Disabling Thresholds

To toggle individual thresholds on or off, use the ID of a threshold to access the corresponding threshold definition on the definitions property of the IGaugeThresholdsConfig object, and set the enabled property on the threshold definition accordingly.

Linear with Thresholds

The steps for enabling/reversing thresholds and omitting/toggling threshold markers and labels is basically the same for the linear gauge as it is for the donut gauge. The only difference lies in the dynamic toggling of the labels (if needed) in which the linear gauge uses the LinearGaugeLabelsPlugin instead of the DonutGaugeLabelsPlugin. If label toggling isn't needed, you can ignore this fact and simply use the GaugeUtil's createChartAssist function which will add the labels plugin to the gauge automatically unless the disableMarkers property on the IGaugeThresholdsConfig is set to true. For instructions on how to dynamically toggle the threshold markers and labels, take a look at the Dynamic Marker Toggling example in the Donut Gauge section above.

Configuring Thickness

The linear gauge supports any thickness, but Nova provides two standard ones (Small and Large) via the StandardLinearGaugeThickness enum. The enum members or any number value can be assigned to the linearThickness property on the IGaugeConfig in order to set the thickness of the gauge. Additionally, the radius of the threshold markers can be specified on the markerRadius property of the IGaugeThresholdsConfig. Although any number value can be used for the marker radius, Nova provides two standard sizes (Small and Large) for this property via the StandardGaugeThresholdMarkerRadius enum.
