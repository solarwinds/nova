Gauges

Gauges can be an effective way to display a single metric value in visual form. And, Nova Charts provides the ability to create two types of gauges (donut and linear) with the assistance of a utility class called GaugeUtil. Three key methods provided by the GaugeUtil get you most of the way towards a functioning gauge:

  • assembleSeriesSet: assembles a gauge series set with all of the standard scales, renderers, accessors, etc. needed for creating a gauge visualization based on the provided IGaugeConfig object and GaugeMode.
  • updateSeriesSet: updates a gauge series set using the provided IGaugeConfig.
  • createChartAssist: can be used to generate a ChartAssist that's pre-configured based on the specified GaugeMode. The ChartAssist handles pre-processing of the data series to a format that can be used for displaying each of the supported gauge modes.

Steps for Creating a Gauge

  1. Create a ChartAssist using GaugeUtil's createChartAssist function.
  2. Update the ChartAssist's associated chart with a series set generated by the GaugeUtil's assembleSeriesSet function which takes an IGaugeConfig object as the first argument and a GaugeMode value for the second argument.
  3. Optionally, at any time after gauge instantiation, you can update the gauge with a new value, max value, and other configuration changes using the GaugeUtil's updateSeriesSet function.

Donut Gauge

A donut gauge is really just a specialized donut chart with data formatted in a way that results in a gauge-like visualization. It's well-suited for use cases in which there's ample room to display a relatively large visualization. For use cases with a more compact display area, consider the linear gauge type whose documentation can be found further down this page.

Thresholds

To add thresholds to a gauge, specify a value of type IGaugeThresholdsConfig on the thresholds property of the IGaugeConfig. For an easy way to assemble an IGaugeThresholdsConfig object, GaugeUtil provides a createStandardThresholdsConfig function. The definitions property of the function's output is automatically populated with a standard set of threshold definitions each of type IGaugeThresholdDef, and the property includes thresholds for "warning" and "critical" levels.

Each threshold definition has an id, value, color, and enabled property. And, the id's are used as the keys of the threshold definitions object to create an id-to-threshold mapping for easy access. The id's for common thresholds are standardized in the StandardGaugeThresholdId enum which has Warning and Critical members. You can either use the createStandardThresholdsConfig convenience function mentioned above or construct your own threshold configuration with as many or few threshold definitions as you need.

Colors

As mentioned above, the return value of the createStandardThresholdsConfig function has a definitions property which contains a standard set of threshold definitions. Each of these definitions has a color property that's used as the threshold's display color when it's triggered. By default, the function sets the warning threshold to yellow and the critical to red. You can modify the display colors for these and any custom thresholds using this property.

Labels

You can configure a gauge to show labels for the threshold markers on mouse hover. To incorporate the labels, add a DonutGaugeLabelsPlugin instance to the chart. If want to provide a custom formatter for the labels, take a look at the label customization example on this page.

Note: The DonutGaugeLabelsPlugin automatically applies a 25px margin to all sides of the gauge in order to provide a reasonable default clearance for the labels. You can specify a custom clearance by setting the clearance property on the plugin constructor's optional config argument to a value of type IAllAround<number>. Alternately, if you'd prefer to set the margins on the grid manually instead of having the labels plugin do it for you, you can set the applyClearance property on the plugin constructor's config argument to false.

Reversed Thresholds

The default behavior of a Nova gauge is to treat quantity values that are greater than or equal to the threshold values as being triggered. If you want to reverse that and have the gauge treat quantity values that are less than or equal to the threshold values as triggered, set the reversed property on the IGaugeThresholdsConfig object to true.

Turning Off the Threshold Markers

To display thresholds without the threshold marker dots, set the disableMarkers property on the IGaugeThresholdsConfig object to true. You can also dynamically disable the threshold labels by setting the disableThresholdLabels property on the label plugin's config object.

Enabling/Disabling Thresholds

To toggle a threshold on or off, use the ID of the threshold to access the corresponding threshold definition on the definitions property of the IGaugeThresholdsConfig object. Set the enabled property on the threshold definition accordingly.

Customizing the Threshold Label Formatter

To customize the format of the labels shown when the mouse is hovered over the gauge, set the labelFormatter property on the IGaugeConfig object with a function of type Formatter<string>. The function should take a string as the raw data argument and return a string with your customized formatting.
