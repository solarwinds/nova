The List schematic provides the ability to create a generic list that can cover a majority of use cases for a list. Its main responsibility is to integrate the functionality of the nui-repeat component with other components commonly used with a list such as nui-selector, nui-search, nui-paginator, and nui-sorter.

Notable Command Line Options

Basic Usage

This example illustrates the output of the CLI command without any modifications. Consumers are able to build upon the generated boilerplate to provide additional functionality depending on their use case. Data manipulations (filtering/pagination/sorting/search) are handled by the ClientSideDataSource service, which can be extended with custom filtering behavior.

ng g @nova-ui/bits:list --name=basic-list --p=app

List with Selection

Selection is facilitated by the ListService and can be enabled by running the CLI command with the selectionMode option set to one of the selection-enabling values listed under Notable Command Line Options section above. The initial selection can be set using the selection input of the nui-repeat. Selection changes can be intercepted using the selectionChange output of the nui-repeat.

ng g @nova-ui/bits:list --name=selection-list --p=app --selectionMode=multi

Search Properties

To specify fields for search, pass a properties array to the setSearchProperties method exposed by the ClientSideDataSource service. If no properties are specified, the search will be performed on all fields.

ng g @nova-ui/bits:list --name=search-list --p=app --selectionMode=none --dataSource=serverSide --pagingMode=pagination --enable-sort=false --enable-search=true

Pagination

This example shows how to add pagination to nui-repeat with a remote data source.

ng g @nova-ui/bits:list --name=paginated-list --p=app --pagingMode=pagination --dataSource=serverSide

Virtual Scroll

This example shows how to add virtual scroll to nui-repeat with a remote data source.

Since the example makes use of search and sorting filters, it's important to note that the viewport manager's paging progress must be reset each time the data is refreshed with new filtered results. The reset invocation can be seen in the virtual-scroll-list.component.ts:doSearch method. ng g @nova-ui/bits:list --name=scrollable-list --p=app --pagingMode=virtualScroll --dataSource=serverSide
