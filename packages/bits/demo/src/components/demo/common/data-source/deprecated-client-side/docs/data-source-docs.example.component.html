Local Filtering Data Source

The main responsibility of a data source is to process one or more filters (e.g. pagination, sorting, search, etc.) from any filtering components before exposing the data to a presentation layer. The LocalFilteringDataSource aims to allow filtering components to be independent from the main presentation component by allowing each of them to communicate to the service how they each individually want to filter the data. The presentation component can then ask for the pre-filtered data via the service.

Filtering components can be registered with the service in the following manner:
  • use this.dataSourceService.componentTree = {{ "{" }} search: {{ "{" }}...{{ "}" }} {{ "}" }}; to set components that will be part of filtering process
  • implement getFilteredData in your own service or use existing from LocalFilteringDataSource
  • use it by subscribing to output and applying newly received data

Visualization of data flow is available here.

Warning: If you are planning on using AoT compilation, avoid directly providing LocalFilteringDataSource. The search service inside LocalFilteringDataSource will not be populated under AoT compilation (JIT compilation will work normally). A simple work around is to create a simple class, have it inherit/extend LocalFilteringDataSource, and then provide that.
