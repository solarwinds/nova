The <nui-combobox-v2> component allows users to select item(s) from a list.

Combobox works best when the options list is big enough and you need to filter it using typeahead. In case your list contains few items or you need to use custom input, consider using Select Component

Required Modules

  • NuiSelectV2Module

Basic Usage

To use <nui-combobox-v2>, wrap each of the options within a <select-v2-option> instance. <select-v2-option> has the value input so that it's unique for each option.

You can use a string or an object as a value. In some cases, e.g. if using Grouped items, it's recommended to use object and provide some id's, because you can have the same option title in the different groups and it's necessary to distinguish them somehow. Also, using string may cause i18n issues as far as the same value is used for model and displaying on the view.

Wrap all of the options inside a <nui-combobox-v2> instance as a content projection for the dropdown.

You can put pretty much anything inside a <nui-combobox-v2> dropdown. For example, if you need the dropdown to display some info text, you can project standard HTML into the dropdown and it will be shown as standard HTML rather than a select option.

Highlighting of the searching string is the default functionality, read more about it in the Typeahead section.

Getting and setting the combobox value

Getting a Value

You can use the following ways to get a value from a combobox component:

  • Access the value property on the component
  • Bind to the valueSelected output - emits the selected value
  • Bind to the valueChanged output - emits after input was changed

Setting a Value

To set a value for a combobox, you can use either the value input or the reactive form control setValue method, to provide an option from the list.

Typeahead

<nui-combobox-v2> comes with built-in typeahead functionality so you can filter options in the list.

To enable highlighting of the search string within the combobox options, you can use the default highlighting functionality provided by the nuiComboboxV2OptionHighlight directive:

<span [nuiComboboxV2OptionHighlight]="item"><span>

For more detail on how to get this working, take a look at the code inside the source expander of the Basic Usage example above.

Note: If you want more customization, you can implement custom highlighting via the HighlightPipe instead of nuiComboboxV2OptionHighlight. Note: You can disable or use custom typeahead behavior if the default behavior doesn't meet your requirements. For that, please see the Custom Typeahead section. As always, before implementing a custom behavior, it's important to consult with UX leadership. This will ensure that the desired behavior conforms to the Nova Design Language.

Options Customization

It's possible to use a custom template for the options in the list. You can use pretty much any layout for the options by putting the template inside a <select-v2-option> instance.

Grouped Options

If you want to group options you can wrap them in a <nui-select-v2-option-group> instance. You can also provide a header for a group (optionally).

To use default styles for a header (as in the example below), apply .nui-select-v2-option-group--header class to the header.

Note: Technically you can use any template instead of, or in addition to, the header.

Using With Reactive Forms

To use nui-combobox-v2 with reactive forms, create a FormControl that represents the form model. Then, bind nui-combobox-v2 to this form element.

Note: It's recommended to use the <nui-form-field> component as a wrapper if you are going to use validation, captions, labels, etc.

Multiselect

Combobox has a multiselect mode. To turn it on, set the multiselect input to true. This allows the template you pass to the inputTemplate input to render the selected items in a special container within the main input.

Note: It's possible to use another component for this feature in place of nui-chips. In fact, you can technically pass any template to inputTemplate.

Creating an Option

You can create new options and add them to the list. To do that, just add the option to the initial list e.g. options.push(option);. To immediately set an added option as the selected value you can use either FormControl.setValue(option); or the value input on the component.

If you want to add options using the Create Option button as in the example below, you should add the button as a content projection along with the options themselves.

Note: You can use the .nui-combobox-v2__create-option class for the Create Option block to have the same styles as the example.

nui-combobox-v2 comes with the following outputs which are useful when implementing the create-option functionality:

  • canCreateOption - emits true when there's no such option already
  • searchEmpty - emits true when there's no option that fits search term in the textbox. Note: This is valid only if the isTypeaheadEnabled property is true (default behavior).

Virtual Scroll

The options dropdown can use virtual scroll. For that, wrap the options list in a cdk-virtual-scroll-viewport. You can adjust virtual scroll container to your needs according to Virtual Scroll API.

Warning: All the items inside a dropdown must be of the same height!

You should set itemSize property of cdk-virtual-scroll-viewport to the exact value of an option height. Otherwise, some features may not work. Note: If using virtual scroll, custom typeahead must be used. See the Custom Typeahead section for how to implement that.

Disabled Mode

It is possible to disable either the entire combobox-v2 or one or more select-v2-options. To do that, set the isDisabled input to true on the appropriate component.

Error State

The isInErrorState input is used to set/change the visual validation state of the component. This can be used in custom situations in which you need to manually indicate that the component is in error state.

Note: If you want to use a combobox as a form field with validation, consider using the nui-form-field component. For information about that, see the Using With Reactive Forms section.

Styling

To style the combobox's dropdown simply use overlayConfig input and fill in its panelClass property with the global CSS class with the desired styles.

Inline Display Mode

To put a combobox in inline display mode, add necessary styles to the component. For example, you can add bootstrap d-inline-flex style.

Value Removal

Combobox has a "remove value" button by default. If you don't want to use it, set the isRemoveValueEnabled input to false.

Note: You should probably add validation as in Reactive Forms section.

Overlay Configuration

To adjust ComboboxComponent's popup overlay properties, such as width, height or other properties, set the overlayConfig input in accordance with the Angular CDK Overlay Config interface.
NOTE: in case you need to set the width using the overlay config make sure you disable the automatic width synchronization, which is enabled by default. This can be done by setting the syncWidth input to false Note: Select-v2 and Combobox-v2 contain default with-popup-styles (you can import it POPUP_STYLES_CLASS from @nova-ui/bits) class which styles the popup according to the Nova Design Language. Setting custom styles through the global classes using the `panelClass` will override the default styles of the overlay. If you want to preserve default overlay styles please consider adding the with-popup-styles class when setting styles using the overlayConfig. Please see the example below.

Custom Typeahead

Combobox comes with default typeahead functionality. Nevertheless, it's possible to use custom typeahead logic. To do so, disable the default typeahead by setting the isTypeaheadEnabled input to false. Custom typeahead involves filtering the options list and updating it manually.

Custom Dropdown Control

Similar to select-v2, the manualDropdownControl boolean input gives you total control over opening and closing the dropdown. When set to true, the default behavior of the dropdown is blocked, and you can provide your own behavior.

Note: The clickOutsideDropdown output can be used to close the dropdown manually on clicks outside both the host element and the dropdown. Note: In order for keyboard navigation work as you open the dropdown you have to manually set focus to the inputElement in case you are triggering the dropdown outside the host element. For that, you can do combobox.inputElement.nativeElement.focus()

Keyboard interaction

<nui-combobox-v2> comes with built-in accessibility features:

  • TAB/Shift+TAB for focusing the component.
  • ARROW UP/ARROW DOWN for navigation inside an option list.
  • PAGE UP/PAGE DOWN for navigation inside the list to top/bottom.
  • ENTER to select an option.

Multiselect only:

  • BACKSPACE to remove a selected item.
  • LEFT to navigate the selected items.
  • RIGHT to navigate the selected items.
